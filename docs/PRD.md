# 產品需求文件 (PRD)

## 1. 專案概述
- **專案名稱**：AI 學習助理平台
- **專案描述**：一個利用 AI 協助學生整理筆記、生成題目與規劃學習進度的智慧學習系統。
- **背景與動機**：學生在面對大量且多樣的學習內容時，常花費過多時間在整理精華重點、準備練習題目，也很難具體掌握自己的弱點。本系統期望將這些重複、耗時的工作交給 AI，讓學習變得更有針對性與效率。
- **目標用戶**：高中生、大學生、準備考試的學習者。
- **核心價值主張**：透過 AI 賦能的自動化筆記與因材施教的測驗，為學習者量身打造專屬的虛擬隨身家教。

## 2. 功能需求

1. **自動整理筆記**
   - 作為 **學生**，我希望 **能上傳講義**，以便 **系統自動為我生成精練的 AI 摘要與筆記**。

2. **AI 出題與測驗（選擇題 / 簡答題）**
   - 作為 **準備考試的學習者**，我希望 **系統能根據上傳或整理好的筆記自動出題**，以便 **我可以在學習後立刻測驗自己的理解程度**。

3. **學習進度追蹤（每日學習分析）**
   - 作為 **學生**，我希望 **系統能記錄並視覺化呈現我的每日學習時數與測驗量**，以便 **我能掌握並調整自己的學習節奏與進度**。

4. **弱點分析與複習建議**
   - 作為 **準備考試的學習者**，我希望 **系統能根據我的歷次測驗結果找出不熟的章節並給予複習建議**，以便 **我能針對弱點進行加強複習**。

5. **語音問答學習（像聊天一樣問問題）**
   - 作為 **大學生**，我希望 **能用對話或語音的方式向系統提問**，以便 **在思考卡關時能獲得像家教般的即時解答與引導**。

6. **多科目整合（數學、英文、程式等）**
   - 作為 **學生**，我希望 **能在同一個平台上建立多種科目的各別專區**，以便 **統一管理我不同領域的學習資源，不讓筆記散落各處**。

## 3. 非功能需求
- **技術限制**：
  - Web 框架：使用 **Flask**
  - 模板引擎：使用 **Jinja2**
  - 資料庫：使用 **SQLite**
- **效能與安全考量**：
  - **回應延遲處理**：呼叫 AI API（如產生筆記、出題）可能需要較長等待時間，前端需實作 Loading 狀態以提升使用者體驗。
  - **檔案上傳安全**：應限制上傳檔案大小及支援格式（如 txt, pdf, png 等），防止惡意拖慢系統或執行檔上傳。
  - **資料隔離與隱私**：使用者的筆記與測驗紀錄屬於私人文件，必須實作穩定的身份驗證方案，確保資料不被輕易越權存取。

## 4. MVP 範圍

- **Must Have (必須有)**
  - 使用者註冊與登入功能。
  - 文字筆記的新增/編輯，及從文字呼叫 AI 生成摘要。
  - 根據筆記內容自動生成單向選擇測驗題並記錄分數。
  - 多科目分類管理（建立不同科目的資料夾/標籤）。
- **Should Have (應該有)**
  - 講義檔案上傳（例如簡單的 PDF 與圖檔分析）轉文字功能。
  - 學習進度追蹤報表（圖表顯示過去一週測驗的紀錄）。
  - 弱點提示與錯題本功能。
- **Nice to Have (加分項)**
  - 即時語音輸入與 AI 語音回覆互動回答功能。
  - 自動規劃學習倒數與進度圖。
